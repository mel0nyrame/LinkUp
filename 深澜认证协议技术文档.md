# 深澜认证协议(Srun Portal Protocol)技术文档

## 一, 核心加密特征

| 特征 | 说明 |
| :---: | :---: |
| 协议头 | `{SRBX1}` 表示使用 XXTEA + 自定义 Base64 加密 |
| Base64 字母表 | `LVoJPiCN2R8G90yg+hmFHuacZ1OWMnrsSTXkYpUq/3dlbfKwv6xztjI7DeBE45QA` |
| Delta 常数 | `0x9E3779B9` (XXTEA 算法魔数) |

## 二, 关键 API 接口

| 接口路径 | 方法 | 功能 |
| :---: | :---: | :---: |
| `/cgi-bin/rad_user_info` | GET | 获取 IP, 查询在线状态 |
| `/cgi-bin/get_challenge` | GET | 获取 Token (32位十六进制随机字符串) |
| `/cgi-bin/srun_portal` | GET | 登录/注销提交 |
| `/v1/srun_portal_online` | GET | 查询在线设备列表 |

## 三, 核心字段概念

| 字段 | 含义 |
| :---: | :---: |
| `ACID` | 接入点 ID (如 1, 2, 5 标识楼栋/区域) |
| `Token/Challenge` | `get_challenge` 返回的 32 位十六进制字符串, 用于加密密码 |
| `HMD5` | `HMAC-MD5(password, token)` 的加密结果 |
| `Info` | `{SRBX1}` 开头的加密字符串 (XXTEA 加密后的 JSON 数据) |
| `Chkstr` | 校验字符串: `token+username+token+hmd5+token+acid+token+ip+token+n+token+type+token+info` |
| `Chksum` | `SHA1(chkstr)` 的结果, 用于防篡改 |

## 四, XXTEA 加密实现要点

- 模式: `ECB` (电子密码本)
- 密钥: `Token` 字符串转 32 位整数数组(小端序)
- Delta: `0x9E3779B9` (JS 中写作 `0x86014019 | 0x183639A0`)
- 轮数: `floor(6 + 52/(n+1))`, n 为数据块数-1
- 填充: 最后一个整数存储原始字符串长度(字节数)
- 字节序: 小端序 (`Little Endian`)

## 五, 自定义 Base64 实现

- 标准表: `ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/`
- 深澜表: `LVoJPiCN2R8G90yg+hmFHuacZ1OWMnrsSTXkYpUq/3dlbfKwv6xztjI7DeBE45QA`
- 实现方式: 先标准 `Base64` 编码, 再按字符映射表替换

## 六, 加密算法细节

| 算法 | 说明 |
| :---: | :---: |
| `HMAC-MD5` | `Key=Token`, `Message=Password`, 输出十六进制小写 |
| `SHA1` | 输入为 ASCII 字符串, 输出十六进制小写 |

## 七, 四步登录流程

- 步骤
  1. 获取 IP: `GET /cgi-bin/rad_user_info`, 返回 `online_ip`
  2. 获取 Token: `GET /cgi-bin/get_challenge?username=xxx&ip=xxx`, 返回 `challenge`
  3. 本地计算: `hmd5=HMAC-MD5(pwd,token)`, `info=XXTEA加密(Base64)`, `chksum=SHA1(chkstr)`
  4. 提交登录: `GET /cgi-bin/srun_portal?action=login&...`, 参数包含 `{MD5}hmd5`, `chksum`, `info` 等

## 八, API 端点详细说明

- `/cgi-bin/rad_user_info` (GET)
  1. 查询在线状态
  2. 返回字段: `error` ("ok"或"not_online_error"), `online_ip`, `sum_bytes`, `sum_seconds`
- `/cgi-bin/get_challenge` (GET)
  1. 获取 Token
  2. 返回字段: `challenge`
- `/cgi-bin/srun_portal` (GET)
  1. 登录/注销
  2. `error` 为 "ok" 表示成功
- `/v1/srun_portal_online` (GET)
  1. 查询在线设备列表
  2. 需携带 `username` 和 `password` 的 md5
- `:8800` 或 `/site` (GET)
  1. 自服务系统(流量, 余额, 设备详情)

## 九, 在线状态验证

- 必须调用 `rad_user_info` 确认登录成功, 仅凭 `srun_portal` 返回不可靠
- 在线标志: `error == "ok"`
- 离线标志: `error == "not_online_error"`
- 设备详情: `online_device_detail` 字段是 JSON 字符串(需二次解析)

## 十, ACID 获取策略

- 首选: 从登录页 URL 参数 `ac_id=` 提取
- 次选: 遍历试探(1-20), 调用 `get_challenge`, 能获取到 Token 即视为有效
- 常见值: `1` (默认), `2`, `5`, `11`, `15`

## 十一, 跨语言实现注意

- 深澜协议中字符串操作基于 `Latin-1(ISO-8859-1)` 编码, 不是 `UTF-8`
- 处理中文用户名时需特别小心(但通常校园网账号是数字/字母)

## 十二, 常见错误码

| 错误码 | 含义 |
| :---: | :---: |
| `E2901` | 密码错误 (检查密码或账号后缀@域) |
| `E2620` | 设备数超限 (需要下线其他设备或等待) |
| `ip_already_online_error` | IP 已被其他账号占用 (先注销再登录) |
| `not_online_error` | 未在线(查询时, 正常状态) |
| `ACID error` | 接入点 ID 错误 (更换 ACID 重试) |
